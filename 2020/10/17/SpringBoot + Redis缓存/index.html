SpringBoot + Redis缓存

Java Spring Redis
Spring
Publish Date:   2020-10-17

Spring 3.1开始对缓存提供支持，核心思想是对方法的缓存，当开发者调用一个方法时，将方法的参数和返回值作为 key/value 缓存起来，当再次调用该方法时，如果缓存中有数据，就直接从缓存中获取，否则再去执行该方法。

Spring并未提供缓存的实现，而是提供了一套缓存API，我们可以自由选择缓存的实现。主流的是 EhCache、Redis。Spring早已将缓存领域统一，无论使用哪种缓存实现，不同的只是缓存配置。

1.创建项目，添加依赖

        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-cache</artifactId>
        </dependency>

        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- test依赖 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
            <exclusions>
                <exclusion>
                    <groupId>org.junit.vintage</groupId>
                    <artifactId>junit-vintage-engine</artifactId>
                </exclusion>
            </exclusions>
        </dependency>

        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-data-redis</artifactId>
            <!-- 去除对lettuce的依赖,因为默认的redis工具是lettuce -->
            <exclusions>
                <exclusion>
                    <groupId>io.lettuce</groupId>
                    <artifactId>lettuce-core</artifactId>
                </exclusion>
            </exclusions>
        </dependency>

        <!--  jedis连接池 -->
        <dependency>
            <groupId>redis.clients</groupId>
            <artifactId>jedis</artifactId>
        </dependency>

2.整合Redis

Redis单机缓存只需要在 application.properties中进行redis配置及缓存配置即可。（要启动redis server）

# 缓存配置
# 配置缓存名称，redis的key都有一个前缀，默认前缀就是“缓存名::”
spring.cache.cache-names=c1,c2
# 缓存有效期，即redis中key的过期时间
spring.cache.redis.time-to-live=1800s

# redis配置
# redis数据库，默认是0号数据库
spring.redis.database=0
spring.redis.host=127.0.0.1
spring.redis.port=6379
# 登录密码，默认为空
spring.redis.password=
# 连接池最大连接数，默认为8
spring.redis.jedis.pool.max-active=8
# 连接池最大空闲连接数
spring.redis.jedis.pool.max-idle=8
# 连接池最大阻塞等待时间，默认为-1，表示没有限制
spring.redis.jedis.pool.max-wait=-1ms
spring.redis.jedis.pool.min-idle=0

3.开启缓存

在项目的入口类上添加 @EnableCaching 注解开启缓存

@SpringBootApplication
@EnableCaching
public class CacheApplication {
    public static void main(String[] args) {
        SpringApplication.run(CacheApplication.class, args);
    }
}

4.创建 Dao

public class Book implements Serializable {
        private Interger id;
        private String name;
        private String author;
        // 省略getter/setter
}
@Repository
// 指明所使用的缓存的名字。也可不写，直接在@Cacheable注解中指明缓存名字
@CacheConfig(cacheNames = "book_cache")
public class BookService {
    // 此注解表示对该方法进行缓存。默认key是方法的参数，value是方法的返回值
    // 当其他类中调用该方法时，首先会根据参数查看缓存中是否有相关数据
    // 如果缓存有，直接使用缓存数据，该方法不会被执行。否则执行该方法，执行成功后将返回值缓存起来。
    // 当前类调用该方法，缓存不会生效
    // @Cacheable(condition="#id%2==0") 表示对id对2取模为0时才进行缓存，否则不缓存。
    @Cacheable
    public Book getBookById(Integer id) {
        System.out.println("getBookById");
        Book book = new Book();
        book.setId(id);
        book.setName("三国演义");
        book.setAuthor("罗贯中");
        return book;
    }

    // 一般用于数据更新方法上，方法每次在执行时都不会检查缓存中是否有数据，而是直接执行方法，将结果缓存。
    // 如果key对应的数据已经被缓存，会覆盖之前的数据，避免再次加载数据时获取到脏数据。
    // 缓存的key为参数book对象中id的值
    @CachePut(key = "#book.id")
    public Book updateBookById(Book book) {
        System.out.println("updateBookById");
        book.setName("三国");
        return book;
    }

    // 一般用于删除方法上，表示移除一个key对应的缓存。
    // 有两个特殊的属性：allEntries、beforInvocation。用于移除缓存数据。默认均为false
    // 例如@CacheEvict(beforeInvocation="false") 在方法执行之后移除缓存中的数据
    // 缓存的key为参数id
    @CacheEvict(key = "#id")
    public void deleteBookById(Integer id) {
        System.out.println("deleteBookById");
    }
}

5.测试

创建测试类，对Service中的方法进行测试

@SpringBootTest
class CacheApplicationTests {

    @Autowired
    BookService bookService;

    @Test
    void contextLoads() {
        bookService.getBookById(1);
        bookService.getBookById(1);
        bookService.deleteBookById(1);
        Book b3 = bookService.getBookById(1);
        System.out.println("b3:" + b3);
        Book b = new Book();
        b.setName("三国演义");
        b.setAuthor("罗贯中");
        b.setId(1);
        bookService.updateBookById(b);
        Book b4 = bookService.getBookById(1);
        System.out.println("b4:" + b4);
    }
}

执行该方法，控制台打印日志如图：

一开始执行了两个查询 getBookById()，但是查询方法只打印了一次，因为第二次使用了缓存。

接下来执行了删除方法，将缓存删除。

之后再次执行查询，因为删除方法已经将缓存删除了。

接下来执行更新方法，更新为 b对象中的数据，不仅更新数据，也更新了缓存。

所以最后的查询方法中，查询方法日志没打印，说明该方法没执行，而是使用了缓存中的数据，而缓存中的数据已经被更新了。
